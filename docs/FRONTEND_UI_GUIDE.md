# NFA-Linux Frontend UI Guide

## Overview

The NFA-Linux frontend is a **dark-mode native forensic dashboard** built with:
- **React 18** + TypeScript
- **Wails v2** for Go backend integration
- **TailwindCSS** with custom cyberpunk purple theme
- **TanStack Virtual** for million-row virtualized tables
- **Three.js / react-force-graph-3d** for 3D network topology
- **Lucide React** for icons

## Key UI Components

### StatsCard
Real-time statistics cards with trend indicators and color-coded severity.

### HexViewer
Full-featured hex editor with:
- Dual hex/ASCII view
- Search (hex and ASCII)
- Byte highlighting for protocol fields
- Click-to-select with offset display
- Copy hex values

### NetworkGraph (3D)
Interactive Three.js visualization:
- Force-directed layout
- Node sizing by traffic volume
- Color-coded by node type
- Animated packet flow particles
- Click to zoom/focus
- Alert indicators on nodes

### Virtualized Tables
All tables use TanStack Virtual for:
- Million-row support
- 60fps scrolling
- Dynamic row heights
- Sticky headers

## Responsive Design

The UI adapts to different screen sizes:
- **Desktop (1920+)**: Full layout with all panels
- **Laptop (1280-1920)**: Condensed stats, collapsible sidebar
- **Tablet (768-1280)**: Stacked layout, hidden secondary panels
- **Mobile**: Not recommended for forensic analysis

## Performance Optimizations

1. **Virtual Scrolling**: Only renders visible rows
2. **Memoization**: React.memo on expensive components
3. **Throttled Updates**: 60fps cap on Wails events
4. **Web Workers**: Heavy computations off main thread
5. **Canvas Rendering**: Charts use Canvas, not SVG
6. **Lazy Loading**: Views loaded on demand
